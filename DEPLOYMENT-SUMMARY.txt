================================================================================
STEP-BY-STEP DEPLOYMENT GUIDE
================================================================================

PHASE 1: DEPLOY BACKEND (5 minutes)
──────────────────────────────────
1. Go to railway.app
2. Sign up with GitHub
3. Create new project → Deploy from GitHub
4. Select vtwo-ops repository
5. Add environment variables:
   - NODE_ENV=production
   - SESSION_SECRET=[generate with openssl rand -base64 32]
   - (FRONTEND_URL will be set later)
6. Click Deploy → Wait for build
7. Copy your API URL (e.g., your-api-xxx.railway.app)

PHASE 2: DEPLOY FRONTEND (2 minutes)
────────────────────────────────────
1. Go to vercel.com
2. Sign up with GitHub
3. Create new project → Import from GitHub
4. Select vtwo-ops repository
5. Set Root Directory: app
6. Add environment variable:
   - VITE_API_URL=[Your Railway API URL from Phase 1]
7. Click Deploy → Wait for build
8. Copy your frontend URL (e.g., your-app.vercel.app)

PHASE 3: CONNECT SERVICES (1 minute)
────────────────────────────────────
1. Go back to railway.app
2. Click your project → Variables
3. Update FRONTEND_URL:
   FRONTEND_URL=https://your-app.vercel.app
4. Click Redeploy → Wait for build

PHASE 4: VERIFY (2 minutes)
───────────────────────────
1. Test backend: curl https://your-api.railway.app/api/health
2. Open frontend: https://your-app.vercel.app
3. Open DevTools (F12) → Console tab
4. Should see no errors
5. Try using the app → Check Network tab
6. API calls should succeed with 200 status

================================================================================
TROUBLESHOOTING
================================================================================

Backend won't build:
  → Check Railway logs for error
  → Fix code in IDE
  → git push origin main (auto-redeploy)

Frontend won't build:
  → Check Vercel logs for error
  → Usually missing environment variable
  → Add VITE_API_URL and redeploy

CORS errors in browser:
  → Check FRONTEND_URL in Railway
  → Make sure it matches your Vercel URL exactly
  → Redeploy backend

API calls return 404:
  → Check VITE_API_URL in Vercel
  → Make sure it matches your Railway URL exactly
  → Redeploy frontend

See docs/MAINTENANCE.md for more troubleshooting
